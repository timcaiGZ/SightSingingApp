## 1. 引言

### 1.1 目的

本文档定义面向民谣吉他进阶学习者的视唱练耳移动应用的功能需求和非功能需求，作为开发团队与指导老师之间的沟通桥梁，明确项目范围、功能边界和质量要求。

### 1.2 范围

本项目是一款 iOS 原生应用，名为 **SightSingingApp**，核心功能包括：

- 三种谱式（五线谱、六线谱、简谱）的展示与切换
- 听力训练（单音、音程、和弦、调式、节奏、旋律）
- 视唱练习与实时音准检测
- 结构化课程学习路径
- 学习进度统计与分析
- 专业乐理知识库

### 1.3 定义与缩写

| 术语 | 定义 |
|------|------|
| 视唱 | 根据乐谱演唱的技能 |
| 练耳 | 通过听觉识别音高、节奏等的技能 |
| 六线谱 | 吉他专用谱表，展示弦号和品位 |
| 简谱 | 用数字1-7表示音高的记谱法 |
| 音分 | 音高最小计量单位，1半音=100音分 |
| 音准偏差 | 演唱音高与目标音高的音分数差异 |

### 1.4 参考资料

| 资料 | 说明 |
|------|------|
| Solfeggio（练耳大师）| App Store 参考应用，通用视唱练耳工具 |
| EarMaster | 专业练耳软件 |
| Apple Human Interface Guidelines | iOS 设计规范 |
| SwiftUI 官方文档 | 框架技术文档 |

### 2.2 设计理念升级（V2.0）

**核心升级**：从 iOS 原生风格 → 深蓝专业主题 + 卡片式布局

| 维度 | V1.0 设计 | V2.0 设计 | 改进说明 |
|------|-----------|-----------|----------|
| **视觉风格** | 纯白背景、iOS原生灰 | 深蓝主色调(#1E3A5F)、卡片布局 | 更专业、更聚焦 |
| **布局方式** | 分隔线列表 | 卡片式布局 | 更现代、更有层次 |
| **进度指示** | 简单圆点 | 彩色圆点 + 圆点数组 | 更直观 |
| **模块标识** | 无 | 左侧彩色条 | 快速识别模块类型 |
| **交互反馈** | 基础 | 实时音准曲线 + 评分 | 更专业 |

### 2.3 参考应用对比

| 维度 | Solfeggio（参考） | SightSingingApp（本项目） |
|------|-------------------|--------------------------|
| 目标用户 | 通用音乐学习者 | 民谣吉他进阶学习者 |
| 默认谱式 | 仅五线谱 | 六线谱 + 简谱 |
| 六线谱支持 | 无 | 真实吉他谱（弦号+品位+指法） |
| Tab结构 | 自由练习/课程/乐理/我的 | 对齐参考应用 |
| 课程结构 | 4层深度 | 对齐，内容适配吉他场景 |
| 视觉风格 | 纯白极简 | 深蓝专业主题 + 卡片布局 |

### 2.4 参考应用核心设计亮点

1. **统一练习容器** — 所有题型共用同一页面结构，减少认知负担
2. **三种交互模式** — 选择题/键盘输入/视唱，各司其职
3. **课程体系** — 4层深度结构化学习路径
4. **专业视觉** — 深蓝主题、卡片式布局、彩色进度指示
5. **实时反馈** — 音准曲线、评分展示、学习趋势图

#### 3.5.3 五线谱设计（参考 Solfeggio）

```
┌─────────────────────────────────────────────┐
│ 𝄞                                      4/4 │
│                                         ─── │
│    ●────●────●────●────●                   │
│    C    D    E    F    G                   │
└─────────────────────────────────────────────┘
```

**设计要点**：
- 5条水平线，线间距10px
- 高音谱号 𝄞 / 低音谱号 𝄢
- 音符直接显示在线或间上
- 支持升降号 ♯ ♭
- 符头（实心/空心）+ 符干 + 符尾表示不同时值

#### 3.5.4 六线谱+简谱设计（参考 Guitar Pro / GTMaker）

```
┌─────────────────────────────────────────────┐
│ 六线谱                                    │
│ 6 ┃ ──────────────────────────────○─────│
│ 5 ┃ ──────────────────────●─3────────────│
│ 4 ┃ ──────────○──────────────────────────│
│ 3 ┃ ──────────●─2─────────────────────────│
│ 2 ┃ ──────────●─0─────────────────────────│
│ 1 ┃─●─3─────────────────────────────────│
│    ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┛
│      0   1   2   3   4   5   6   7        │
├─────────────────────────────────────────────┤
│ 简谱                                        │
│  G      4      3      2      1            │
│ Sol    Fa     Mi     Re     Do            │
└─────────────────────────────────────────────┘
```

**六线谱设计要点**：
- 6条横线代表吉他6根弦（1=最细高音弦，6=最粗低音弦）
- 弦号标注在左侧
- 品位数字直接标在弦上
- 空弦用 ○ 圆圈标记
- 技法符号：h击弦、p勾弦、/滑音、b推弦、~揉弦、x哑音
- 击弦/勾弦用弧线连接
- 可选显示和弦框图

**简谱设计要点**：
- 数字1-7表示音高
- 右侧 ' 表示高八度（如1' = 高音Do）
- 升降号 ♯ ♭ 写在数字左侧
- 时值用竖线表示
- 下方标注唱名便于跟唱

#### 3.5.5 谱式布局

- **六线谱+简谱组合**：上下排列
  - 六线谱在上（演奏指法）
  - 简谱在下（唱名标注）
  - 两者一一对应，方便对照学习

### 3.9 三种交互模式

| 模式 | 适用题型 | 界面特征 |
|------|----------|----------|
| **选择题** (multipleChoice) | 单音辨认、音程比较、和弦辨认、调式辨认 | 灰色"请选择" + 白色选项列表 |
| **键盘输入** (keyboardInput) | 单音/音程/和弦/旋律听写 | 升降号列 + CDEFGAB键 + 功能键 |
| **视唱** (sightSinging) | 单音视唱、旋律视唱 | 竖直刻度尺 + 实时游标 + 评分反馈 |

**ExerciseMode 枚举**：
```swift
enum ExerciseMode: String, CaseIterable {
    case multipleChoice = "选择题"
    case keyboardInput = "键盘输入"
    case sightSinging = "视唱"
}
```

- 弦线粗细：1弦最细，6弦最粗
